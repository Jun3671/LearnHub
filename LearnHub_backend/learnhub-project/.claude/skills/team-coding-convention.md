# 팀 코딩 컨벤션

## 네이밍 규칙

- 클래스명: PascalCase
- 메서드명: camelCase
- 상수: UPPER_SNAKE_CASE
- 패키지명: 소문자, 언더스코어 없음

## 메서드 작성 규칙

- 메서드는 한 가지 일만 한다
- 메서드 길이는 20줄을 넘지 않는다
- 파라미터는 3개 이하로 유지한다

## 주석 규칙

- 공개 API에는 반드시 Javadoc 작성
- 복잡한 비즈니스 로직에는 이유를 설명하는 주석 추가
- TODO 주석에는 담당자와 날짜 포함

## 테스트 규칙

- 테스트 메서드명은 한글로 작성 가능
- Given-When-Then 패턴 사용
- 테스트당 하나의 검증만
